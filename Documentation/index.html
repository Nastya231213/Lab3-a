Послідовний: 125 мілісекунд
Паралельний: 248 мілісекунд

Послідовний BFS: 0 мілісекунд
Паралельний BFS: 0 мілісекунд
Граф, використаний у тестах, має 6 вузлів і наступні ребра: (0-1), (0-2), (1-3), (1-4), (2-4), (3-4), (3-5), (4-5). За результатами виконання тестів можна зробити висновок, що обидва алгоритми були дуже швидкими в цьому конкретному випадку.

3.Dijkstra algorithm

За результатами виконання тестів, я отримала наступні часи виконання для послідовного і паралельного алгоритмів Дейкстри:

Для графа з 100 вузлами:

Послідовний: 0 мілісекунд
Паралельний: 10 мілісекунди

Для графа з 10000 вузлами:

Послідовний: 285 мілісекунд
Паралельний: 1385 мілісекунд
На основі наведених результатів можна зробити висновок, що паралельний алгоритм Дейкстри не завжди забезпечує прискорення порівняно з послідовним. Для графів з невеликою кількістю вузлів (100 вузлів) паралельний алгоритм виконується повільніше, ніж послідовний. Це може бути пов'язано з накладними витратами на створення та управління потоками. Для графів з великою кількістю вузлів (10000 вузлів) паралельний алгоритм також показує гірший час виконання, що може вказувати на неефективну реалізацію або недостатню кількість паралельних обчислювальних ресурсів для такого розміру задачі.

4.Mergesort algorithm

Для масива з 100 елементів:

Послідовний: 0 мілісекунд
Паралельний: 5 мілісекунд

Для масива з 100000 елементів:

Послідовний: 57 мілісекунд
Паралельний: 47 мілісекунд
Отже,паралельний алгоритм сортування злиттям демонструє кращу продуктивність на масиві з 100000 елементів порівняно з послідовним алгоритмом. Однак, на масиві з 100 елементів вигода від використання паралельного алгоритму не так велика через додаткові витрати на створення та управління потоками.

5.Quicksort algorithm

За результатами виконання тестів, я отримала наступні часи виконання для послідовного і паралельного алгоритмів швидкого сортування:

Для масива з 100 елементів:

Послідовний: 0 мілісекунд
Паралельний: 0 мілісекунд

Для масива з 100000 елементів:

Послідовний: 14 мілісекунд
Паралельний: 6 мілісекунд
Отже,результати тестування показують, що для невеликих масивів (100 елементів) обидва алгоритми працюють практично миттєво, без помітних відмінностей у часі виконання. Однак, для більших масивів (100000 елементів) паралельний алгоритм QuickSort демонструє значну перевагу у швидкості виконання порівняно з послідовним алгоритмом.
