Паралельний BFS: 88 мілісекунди
Для тестування були виконані вимірювання часу на послідовному та паралельному алгоритмах Флойда-Уоршелла з використанням графа розміром 100 вузлів. Послідовний: 0 мілісекунда
Паралельний: 330 мілісекунди
Отже,за результатами тестів, виконаних на графі розміром 100 вузлів, паралельний алгоритм Дейкстри виконується приблизно за 330 мілісекунд, тоді як послідовний алгоритм виконується майже миттєво, за 0 мілісекунд. Очевидно, що у цьому конкретному випадку послідовна версія працює швидше, що може бути пов'язано з відносно невеликим розміром графу та накладними витратами паралельного виконання. (4-5).

4.Mergesort algorithm

За результатами виконання тестів, я отримала наступні часи виконання для послідовного і паралельного BFS:
Послідовний: 1 мілісекунда
Паралельний: 35 мілісекунди
Отже,для масивів розміром 100 паралельне сортування злиттям не є оптимальним вибором через високу накладні витрати, що значно збільшують час виконання у порівнянні з послідовним сортуванням. Паралельне сортування має сенс використовувати для значно більших масивів, де переваги мультипоточності будуть більш очевидними і виправдають накладні витрати.

5.Quicksort algorithm

За результатами виконання тестів, я отримала наступні часи виконання для послідовного і паралельного BFS:

Для масива з 100 елементів:

Послідовний: 0 мілісекунда
Паралельний: 0 мілісекунди
Для масива з 100000 елементів:

Послідовний: 14 мілісекунда
Паралельний: 6 мілісекунди
Отжерезультати тестування показують, що для невеликих масивів (100 елементів) обидва алгоритми працюють практично миттєво, без помітних відмінностей у часі виконання. Однак, для більших масивів (100000 елементів) паралельний алгоритм QuickSort демонструє значну перевагу у швидкості виконання порівняно з послідовним алгоритмом.
